# 微信小程序数据库说明

## 数据库类型

**微信小程序云开发数据库 (WeChat Cloud Development Database)**

- **类型**: NoSQL 文档型数据库
- **位置**: 微信云开发环境 (cloud database)
- **配置**: `cloud.DYNAMIC_CURRENT_ENV` (当前云环境)
- **访问方式**: 通过云函数 (cloudfunctions) 进行数据库操作

## 各集合详细说明

### 1. activities (活动管理)
**用途**: 存储所有活动信息，包括普通活动、奖励领取活动、诗饼活动等

**主要字段**:
- `_id`: 活动 ID
- `title`: 活动标题
- `activityType`: 活动类型 (`normal` / `rewardClaim`)
- `status`: 状态 (`draft` / `published` / `ended`)
- `registrations`: 报名用户列表 `[{openid, createdAt, attended}]`
- `registrationCount`: 报名人数
- `startDate` / `endDate`: 活动日期
- `rewardMonthKey`: 奖励关联的月份 (格式：YYYY-MM)
- `sortTime`: 排序时间戳

**相关操作**: 创建、查询、报名、取消报名、标记出席

---

### 2. reading_logs (阅读打卡记录) ⭐
**用途**: **用户阅读打卡内容存储在此集合**

**主要字段**:
- `_id`: 记录 ID
- `openid`: 用户 openid
- `bookTitle`: 书名
- `title` / `contentTitle`: 阅读内容标题
- `author`: 作者
- `duration`: 阅读时长 (分钟)
- `pagesOrChapter`: 阅读页数或章节
- `insight`: 阅读感悟
- `excerpt`: 摘抄内容
- `images`: 图片数组
- `activityId`: 关联活动 ID
- `monthKey`: 月份 (YYYY-MM)
- `dayKey`: 日期 (YYYY-MM-DD)
- `giftEligible`: 是否获得月度礼品资格
- `createdAt`: 创建时间

**相关操作**: 提交打卡、更新打卡、查询个人记录

---

### 3. reading_incentive_rules (阅读激励规则)
**用途**: 配置阅读打卡激励活动规则

**主要字段**:
- `_id`: 规则 ID
- `activityId`: 关联活动 ID
- `thresholdType`: 阈值类型 (`consecutive` 连续 / `accumulated` 累计)
- `thresholdValue`: 目标天数
- `requireOfflineAttendance`: 是否要求线下出席
- `effectMode`: 生效模式 (`immediate` / `scheduled`)
- `effectAt`: 生效时间
- `isActive`: 是否激活
- `createdAt` / `updatedAt`: 时间戳

---

### 4. reward_records (奖励发放记录)
**用途**: 记录用户奖励领取情况

**主要字段**:
- `openid`: 用户 openid
- `monthKey`: 月份
- `rewardType`: 奖励类型
- `status`: 状态 (`pending` / `claimed` / `cancelled`)
- `claimedAt`: 领取时间

---

### 5. users (用户信息)
**用途**: 存储用户基本信息和权限

**主要字段**:
- `_id`: 用户 ID
- `openid`: 微信 openid
- `nickName`: 昵称
- `avatarUrl`: 头像 URL
- `role`: 角色 (`visitor` / `member` / `admin`)
- `status`: 状态 (`pending` / `approved` / `rejected`)
- `phone`: 手机号
- `permissions`: 权限配置

---

### 6. applications (入会申请)
**用途**: 用户申请成为会员的记录

**主要字段**:
- `openid`: 申请人 openid
- `nickName`: 昵称
- `phone`: 手机号
- `introduction`: 介绍
- `status`: 审核状态 (`pending` / `approved` / `rejected`)
- `reviewedAt`: 审核时间
- `createdAt`: 申请时间

---

### 7. notices (公告通知)
**用途**: 站内公告和通知

**主要字段**:
- `title`: 标题
- `content`: 内容
- `priority`: 优先级
- `status`: 状态 (`draft` / `published`)
- `isLatest`: 是否最新公告
- `createdAt`: 创建时间

## 数据库访问示例

```javascript
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

// 查询
db.collection('reading_logs').where({ openid, monthKey }).get()

// 新增
db.collection('reading_logs').add({ data: {...} })

// 更新
db.collection('reading_logs').doc(recordId).update({ data: {...} })

// 删除
db.collection('reading_logs').doc(recordId).remove()
```

## 总结

**阅读打卡记录存储在 `reading_logs` 集合中**，该集合位于微信小程序云开发数据库中。

所有数据库操作都通过云函数 (cloudfunctions) 进行，前端不直接访问数据库，确保数据安全。
